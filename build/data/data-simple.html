A list can be empty, singleton, or many …

Elm forces you to handle all cases. A custom type can sometimes be a better option, but only if there’s a valid reason for using it (otherwise, just use a list!):

  1. A `[“non-empty”] list
  2. A Success a server response
  3. Impossible states

Invalid reasons:

  1. Nothing and [] are essentially the same!

Random.Uniform is a good example of a custom type being better than a list: a default value for an empty list.
