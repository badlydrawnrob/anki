What is the disadvantage of using nested state?

Nested records

``

type Dressing
  = Caesar
  | OliveOil

type DressingMsg
  = SetDressing Dressing

type Msg
  = DressingMsg DressingMsg

type alias Salad =
  { dressing : Dressing }

init =
  { salad = { dressing = Caesar } }
updateDressing : DressingMsg -> Salad -> Salad
updateDressing msg salad =
    case msg of
        SetDressing dressing ->
            { salad | dressing = dressing }

update : Msg -> Model -> Model
update msg model =
  case msg of
    DressingMsg dressingMsg ->
      { model
        | salad =
          updateDressing dressingMsg model.salad
      }

We’ve got to create a nested function to update our nested record which can get quite complicated and ugly if you go more than one level deep. @rtfeldman uses a simple model.form record and an anonymous function to update it.

You can also use extensible records to narrow the types, which doesn’t require nesting records.
