When we only need a small function, we don’t need to name it! Our example above uses lambda style notation: \_. In fact, you can think about currying in this way, too. We could write a multiply function in a couple of ways:

-- Regular style
multiply x y = x * y
-- Lambda style
multiply = \x -> \y -> x * y
-- Curry either one
add1 : Int -> Int
add1 = multiply 1

That’s essentially all currying is. Both functions (under the hood) essentially take one argument at a time, returning a function that takes the next argument, and so on!
